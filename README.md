# Bug Reflection

Bug/问题反思复盘 skill。每次在测试或使用中发现问题时，触发复盘分析：根因定位 + 漏测原因分类（PRD 缺口/代码缺陷/测试缺口/流程缺口）+ 对应的机制和流程改进。

## 目标

不是「修这个 bug」，是**找到系统漏洞，堵上它，让同类问题不再出现**。

## 复盘框架

1. **描述问题** — 现象 + 预期 + 复现步骤
2. **5 Whys 根因分析** — 逐层追问到可操作的根本原因
3. **漏测分类** — 从四个维度判断为什么没被拦住：
   - PRD 缺口：需求没写清楚、边界条件遗漏
   - 代码缺陷：逻辑遗漏、边界条件、数据契约不一致
   - 测试缺口：不知道要测 / 知道但没写
   - 流程缺口：换个人写也会踩同样的坑
4. **改进措施** — 每项对应具体文件、配置或流程变更
5. **执行与验证** — 修代码 → 补测试 → 更新文档

## 核心纪律

- **问题不分大小**：低级错误重复出现说明机制有问题
- **不追责，追机制**：某个人错过，换个人也会错过才是真问题
- **改进措施必须有产出**：废除「以后注意」，必须对应具体变更

## 使用方式

作为 Cowork slash command:

```
/bug-reflection
```

## 输出

复盘结论以结构化条目写入项目 memory/ 目录，可回溯。

## 安装

```bash
# Cowork 平台
mcp__cowork__save_skill with name: bug-reflection

# 或直接复制 SKILL.md 到 skills 目录
cp SKILL.md /path/to/skills/bug-reflection/
```
